Overview

The App Installer is a comprehensive PowerShell application installer system designed for automated Windows setup. It provides both a modern graphical user interface (GUI) and a traditional command-line interface (CLI) for installing and managing 212 essential applications with real-time status tracking, version detection, and centralized logging.

Features

GUI Features (install-gui.ps1)

  • Modern Windows Forms Interface with responsive design and DPI scaling.
  • Responsive GUI Helper loaded from GitHub for multi-monitor and high-DPI support.
  • Generic Logging Module with monthly rotation and markdown format.
  • Real-Time Search & Filter across 271 applications by name, category, or description.
  • Real-Time Progress Tracking with dynamic progress bar and counters.
  • Category Organization into multiple logical groups for easier browsing.
  • Checkbox Selection for individual apps, all apps, or only missing apps.
  • Export/Import Profiles for saving and restoring selections.
  • Uninstall Applications with confirmation and progress tracking.
  • HTML Console Output with color-coded logs.
  • Marketing Information and service details in the right HTML panel.

Core Features (Both GUI & CLI)

  • Version Detection via winget and registry checks.
  • Selective Installation of individual, all, or only missing applications.
  • Centralized Logging to monthly markdown logs under %USERPROFILE%\myTech.Today\logs\.
  • winget Integration for 90+ packages.
  • Custom Installers for applications not available via winget.
  • Comprehensive Error Handling with 100+ winget error code mappings.
  • Silent Installation wherever possible.
  • Administrator Privileges with elevation checks and guidance.
  • Automatic Shortcuts - Creates Start Menu shortcuts for web-based apps (Chrome Remote Desktop).

Usage (Summary)

GUI Mode (install-gui.ps1)

Launch the graphical interface from an elevated PowerShell prompt:

.\install-gui.ps1

The GUI exposes search and filter, category grouping, per-application status, and buttons for refreshing status, selecting all or missing apps, exporting/importing profiles, installing, uninstalling, and exiting.

Command-Line Mode (install.ps1)

.\install.ps1

The CLI provides an interactive menu to install individual apps, all apps, or only missing apps, export/import profiles, uninstall selections, and show or refresh status. It also supports command-line parameters like -Action InstallAll, -Action InstallMissing, and -Action Status.

Export/Import Configuration Profiles

Both the GUI and CLI support exporting and importing application selection profiles. This is useful for:

  • Backing up your application selections.
  • Deploying the same set of applications to multiple machines.
  • Sharing configurations with team members.
  • Standardizing installations across an organization.

Requirements

  • Operating System: Windows 10 (1809+), Windows 11, Windows Server 2016+
  • PowerShell: 5.1 or later (PowerShell 7.2+ recommended)
  • Privileges: Administrator rights required (see Administrator & Antivirus Notes below)
  • winget: Windows Package Manager (recommended, installed by default on Windows 11)
  • .NET Framework: 4.7.2 or later (for GUI version)
    • Note: The GUI installer (install-gui.ps1) will automatically detect and offer to install .NET Framework 4.8 if not present or if an older version is detected.

Administrator & Antivirus Notes

Always run elevated:

  • When using the PowerShell scripts (install-gui.ps1, install.ps1): start PowerShell as Administrator and run the scripts from an elevated session.
  • When using the packaged EXE (for example, install-GUI.exe built via IEXPRESS): allow the UAC elevation prompt and run the EXE as Administrator.

Why elevation is required:

  • Installing software, managing services, and modifying system-wide settings (registry, Program Files, etc.) requires administrative rights.
  • Some installers and winget operations will fail or behave unpredictably without elevation.

Antivirus / SmartScreen behavior:

  • Because the EXE is a custom IEXPRESS self-extracting package, many antivirus products and/or Windows SmartScreen may flag, warn about, or quarantine the EXE on first run.
  • If you trust the source (this repository and your own build), you may need to:
    • Unblock the EXE in the file properties dialog.
    • Mark it as allowed in your antivirus / endpoint protection.
    • Choose "More info" â†’ "Run anyway" if SmartScreen shows a warning.

Recommendation:

  • Prefer running the raw PowerShell script (install-gui.ps1) from an elevated PowerShell session when testing or developing.
  • Use the EXE for deployment/hand-off scenarios, with the expectation that security software may require manual approval.

Basic Installation (Script-Based)

  1. Clone or download the repository.
  2. Open PowerShell as Administrator and change to the app_installer directory.
  3. Run the GUI installer: 
    .\\install-gui.ps1
  4. Or run the CLI installer: 
    .\\install.ps1

Using the Packaged EXE

  1. Build or obtain install-GUI.exe (IEXPRESS package).
  2. Right-click the EXE and choose Run as administrator.
  3. If antivirus or SmartScreen warns or blocks the file, review the warning and, if you trust this tool, allow or unblock it as appropriate.
