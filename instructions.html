üìñ What is App Installer?

The App Installer is a comprehensive PowerShell-based application management system that simplifies the process of installing, updating, and managing software on your computer. Whether you're setting up a new machine, reinstalling Windows, or managing multiple computers, this tool saves you hours of manual work.

Key Benefits:
  • Save Time: Install dozens of applications with just a few clicks
  • Cross-Platform: Works on Windows, macOS, and Linux
  • 270+ Applications: Browsers, development tools, media software, utilities, and more
  • Smart Detection: Automatically detects what's already installed
  • Profile Support: Save and reuse your favorite application sets

Windows (Full GUI Support)

üì• Getting Started

Step 1: Download the App Installer

  1. Visit the GitHub repository
  2. Click the green "Code" button
  3. Select "Download ZIP" to download the entire repository
  4. Extract the ZIP file to a location on your computer (e.g., C:\Tools\app_installer)
üí° Tip: If you have Git installed, you can clone the repository instead: 
git clone https://github.com/mytech-today-now/app_installer.git

Step 2: Open PowerShell with Administrator Rights

‚ö†Ô∏è Important: Administrator privileges are required to install software, modify system settings, and access protected directories.

On Windows:

  1. Press Windows + X on your keyboard
  2. Select "Windows PowerShell (Admin)" or "Terminal (Admin)"
  3. Click "Yes" when prompted by User Account Control (UAC)

Alternative method:

  1. Search for "PowerShell" in the Start menu
  2. Right-click on "Windows PowerShell"
  3. Select "Run as administrator"

Step 3: Navigate to the App Installer Directory

In the PowerShell window, navigate to where you extracted the files:

cd C:\Tools\app_installer

Replace C:\Tools\app_installer with your actual path.

Step 4: Set Execution Policy (Windows Only, First Time)

If this is your first time running PowerShell scripts, you may need to allow script execution:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
What does this do? This command allows you to run locally created scripts while still requiring downloaded scripts to be signed by a trusted publisher. This is a one-time setup.

üéØ Using the App Installer

GUI Mode (Windows Only)

Launch the graphical interface:

.\app-installer.ps1
‚úÖ GUI Features:
  • Console/Instructions Toggle: Click the navigation links at the top to switch between the console output and these instructions
  • Search & Filter: Real-time search across 270+ applications by name, category, or description
  • Category Organization: Applications grouped into logical categories (Browsers, Development, Media, etc.)
  • Smart Selection: Check individual apps, select all, or select only missing applications
  • Progress Tracking: Real-time installation progress with detailed logging
  • Profile Management: Save and load application selection profiles
  • Version Detection: Automatically detects installed applications and their versions

Using the GUI Interface:

  1. Browse Applications: Scroll through the list or use the search box to find specific apps
  2. Select Applications:
    • Click checkboxes next to individual applications
    • Click "Select All" to select everything
    • Click "Select Missing" to select only apps not currently installed
  3. View Instructions: Click the "Instructions" link at the top to view this guide within the app
  4. Install: Click "Install Selected" to begin installation
  5. Monitor Progress: Watch the console output and progress bars for real-time status
  6. Switch Views: Click "Console" to return to the installation output at any time

CLI Mode (All Platforms)

For macOS, Linux, or if you prefer command-line on Windows:

.\app-installer-cli.ps1
üìù Note: On macOS and Linux, running .\app-installer.ps1 will automatically launch the CLI version.

CLI Menu Options:

  1. Install Individual Apps: Select from a numbered list of applications
  2. Install All Apps: Install every available application
  3. Install Missing Apps: Install only applications not currently detected
  4. Show Status: Display which applications are installed and their versions
  5. Export/Import Profiles: Save or load application selection sets
  6. Uninstall Apps: Remove selected applications

Command-Line Parameters (Advanced)

You can automate installations using command-line parameters:

# Install all applications
.\app-installer-cli.ps1 -Action InstallAll

# Install only missing applications
.\app-installer-cli.ps1 -Action InstallMissing

# Show installation status
.\app-installer-cli.ps1 -Action Status

# Force CLI mode on Windows
.\app-installer.ps1 -ForceCLI

üíæ Profile Management

Profiles allow you to save your application selections and reuse them across multiple computers or share them with others.

Exporting a Profile:

  1. Select the applications you want in your profile
  2. Click "Export Profile" (GUI) or select the export option (CLI)

  2. Browse to your saved profile file
  3. The application selections are automatically loaded
  4. Click "Install Selected" to install the profile's applications

üîß Troubleshooting

Common Issues and Solutions

1. "Script cannot be loaded because running scripts is disabled"

Problem: PowerShell's execution policy prevents running scripts.

Solution:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then try running the script again.

2. "Access Denied" or "Administrator privileges required"

Problem: The script is not running with administrator rights.

Solution:

  • Close PowerShell
  • Right-click PowerShell and select "Run as administrator"
  • Navigate back to the app installer directory and run the script again

3. "winget not found" or "winget command not recognized"

Problem: Windows Package Manager (winget) is not installed.

Solution for Windows 11: winget should be pre-installed. Try updating the App Installer from the Microsoft Store.

Solution for Windows 10:

  1. Open Microsoft Store
  2. Search for "App Installer"
  3. Click "Update" or "Get"
  4. Restart PowerShell and try again

Alternatively, the script will attempt to install winget automatically on Windows 10.

4. ".NET Framework version too old" (GUI only)

Problem: The GUI requires .NET Framework 4.7.2 or later.

Solution: The script will automatically detect this and offer to install .NET Framework 4.8. Accept the installation and restart your computer when prompted.

5. "Installation failed" for specific applications

Possible causes:
  • Internet connection issues
  • Application no longer available in winget repository
  • Antivirus blocking the installation
  • Insufficient disk space

Solutions:

  • Check your internet connection
  • Temporarily disable antivirus (if you trust the source)
  • Check the console output for specific error messages
  • Try installing the application manually to see if there's a system-specific issue
  • Check the log files in %USERPROFILE%\myTech.Today\logs\

6. SmartScreen or Antivirus warnings

Problem: Windows SmartScreen or antivirus software flags the script or downloaded installers.

Solution:

  • This is normal for PowerShell scripts and newly downloaded executables
  • If you trust the source (official GitHub repository), click "More info" then "Run anyway"
  • You can also right-click the script file, select "Properties", and check "Unblock" at the bottom
  • For antivirus, add an exception for the app_installer directory

7. macOS/Linux: "GUI mode not available"

This is expected behavior: The GUI uses Windows Forms which is only available on Windows.

Solution: The script automatically launches the CLI version. This is the intended cross-platform experience.

8. Applications not detected as installed

Problem: The script doesn't recognize applications you've already installed.

Possible reasons:

  • Application was installed manually (not via winget)
  • Application was installed to a non-standard location
  • Registry entries are missing or non-standard

Solution: This won't cause problems. The script will attempt to install, and winget will typically detect the existing installation and skip it or upgrade it.

‚ùì Frequently Asked Questions

Q: Is this tool safe to use?

A: Yes! The tool is open-source and available on GitHub. It uses official package managers (winget, Homebrew, etc.) to download applications from their official sources. You can review the source code at any time.

Q: Can I customize which applications are available?

A: Yes! The application list is defined in the script. Advanced users can modify the $script:Applications array to add or remove applications.

Q: Will this work on Windows 7 or 8?

A: No. Windows 10 (1809 or later) is required because the tool relies on winget and modern PowerShell features.

Q: Can I use this in a corporate environment?

A: Yes! Many users deploy this tool for standardizing workstation setups. However, check with your IT department first, as some organizations have policies about software installation tools.

Q: How do I update the App Installer itself?

A: Download the latest version from the GitHub repository or use git pull if you cloned the repository.

Q: Where are the log files stored?

A: Logs are stored in %USERPROFILE%\myTech.Today\logs\ (Windows) or ~/myTech.Today/logs/ (macOS/Linux). Logs are organized by month in markdown format.

Q: Can I run this without administrator rights?

A: No. Installing software requires administrator/root privileges on all operating systems.
